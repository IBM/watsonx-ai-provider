v2.0.0 — Targets Vercel AI SDK v6, drops the `@ibm-cloud/watsonx-ai` SDK in favor of direct REST calls, and aligns the implementation with the Mistral provider in the AI SDK.

### Breaking changes

- **Vercel AI SDK v6 required** (was v4). Peer dep `ai ^6.0.0`.
- **`@ibm-cloud/watsonx-ai` SDK no longer a dependency.** Provider hits `/ml/v1/text/chat`, `/chat_stream`, and `/embeddings` directly via `postJsonToApi`.
- **IAM-only authentication.** `WATSONX_AI_AUTH_TYPE` is no longer read; Bearer / Basic / CP4D modes are gone.
- **`LanguageModelV3`** spec (was V2). Constructor signature: `new WatsonxChatLanguageModel(modelId, config)` — the `settings` argument is removed.
- **`WatsonxChatSettings` removed.** Standard generation params (`temperature`, `topP`, `topK`, `maxOutputTokens`, `stopSequences`) flow through `LanguageModelV3CallOptions`. wx-specific knobs live under `providerOptions.watsonx` (validated by the new `watsonxLanguageModelOptions` zod schema).
- **`provider` strings differentiated**: `'watsonx.chat'` vs `'watsonx.embedding'` (was `'watsonx'` for both).
- **`DEFAULT_MAX_TOKENS` removed** — `maxOutputTokens` passes through; wx.ai's server default applies when unset. v1 silently capped at 1024.
- **`zod` peer dep tightened to `^4.0.0`** (uses `.loose()`).
- **Node engines bumped to `>=20.3.0`** (uses `AbortSignal.any`).
- **Empty user-message file parts** now throw `UnsupportedFunctionalityError` instead of being silently dropped.
- **`createWatsonxProvider`** marked `@deprecated`; alias still exported for back-compat.

### Added

- **Vision / image input** — vision models accept https image URLs directly via `image/*` allowlist on `supportedUrls`.
- **Embedding models** via `provider.embeddingModel(...)` with preflight `TooManyEmbeddingValuesForCallError` for batches over 100.
- **IAM token caching** with auto-refresh and one-shot 401 retry.
- **`providerOptions.watsonx`**:
  - `timeLimit` — server-side wall-clock cap (ms)
  - `parallelToolCalls` — controls concurrent tool calls
  - `reasoningEffort` — `'low'` / `'medium'` / `'high'` for reasoning-capable models. Verified on `openai/gpt-oss-120b` (HIGH ~3× the reasoning of LOW).
- **Custom `fetch` / `generateId` injection** via `WatsonxProviderSettings` — for testing, proxying, deterministic IDs.
- **Default `watsonx` instance** — `import { watsonx } from 'watsonx-ai-provider'` reads env vars automatically.
- **`responseFormat`** — JSON mode and JSON schema.
- **`tool.strict`** flag forwarded when present.
- **`includeRawChunks`** call option emits `{ type: 'raw' }` stream parts.
- **Stream anomaly diagnostics**: always-on warning when wx.ai bills `completion_tokens > 0` but never streams content (known mistral-medium-2505 fingerprint); opt-in raw-chunk trace via `WATSONX_DEBUG_STREAM=1`.

### Fixed

- **gpt-oss tool-call finish reason** — gpt-oss reports `'stop'` instead of `'tool_calls'` for tool responses; provider now overrides correctly.
- **`mistralai/pixtral-12b`** typo (was `pixtrail-12b`).
- **`options.headers`** now forwarded on chat paths (was only honored on embedding).
- **`tool-input-start`** emitted exactly once per tool call (was re-firing on every delta carrying `function.name`).

### Known limitation

- **`mistralai/mistral-medium-2505` streaming + tool calls** is broken on wx.ai's side — the model generates tokens (billed) but never transmits them via SSE deltas. Non-streaming `generateText` works. Use `openai/gpt-oss-120b`, `meta-llama/llama-3-3-70b-instruct`, or `ibm/granite-4-h-small` for streaming tool-call workloads.
